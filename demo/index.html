mangoPicker Demo

Date, Time, พ.ศ., Range, Inline

หน้านี้รวมตัวอย่างใช้งานหลักของ mangoPicker แบบพร้อมใช้งานจริงจากไฟล์ build ใน ../dist รองรับทั้งการ init ด้วย JavaScript options และตั้งค่าจาก HTML ผ่าน data-mangopicker-* ทุกตัวอย่างมี live preview และมีโค้ดซ่อนไว้ก่อน กด แสดงโค้ด แล้วจึงคัดลอกไปใช้ต่อได้ทันที

ใช้ไฟล์จริงจาก dist/mangopicker.js ภาษาไทยจาก dist/i18n/th.js รองรับ data-mangopicker-* หน้า demo ใช้ Bootstrap 5 เป็นฐาน License: MIT
Boilerplate

ชุดเริ่มต้นที่คัดลอกไปใช้ได้ทั้งหน้า

ถ้าจะเริ่มจากศูนย์ ให้คัดลอกชุดนี้ก่อน แล้วค่อยเลือก snippet ของแต่ละเคสด้านล่างไปใส่เพิ่ม

จุดนี้ไม่ใช่ live picker แต่เป็น snippet เต็มที่มี link, script, และตัวอย่าง init พร้อมใช้งาน ถ้าไฟล์ของคุณไม่ได้อยู่ในโฟลเดอร์ demo/ ให้ปรับ path ของ dist ตามตำแหน่งจริง

Date Range

เลือกเป็นช่วงวันที่

เปิด range: true เพื่อให้คลิกวันเริ่มและวันสิ้นสุดใน input เดียว

ตัวอย่างนี้ใช้ range_separator: " ถึง " และเมื่อเลือกครบจะเก็บค่าใน input เป็นวันเริ่มถึงวันสิ้นสุด

12h + Seconds

เวลาแบบ AM/PM พร้อมวินาที

ใช้ format: "h:i:s A" เมื่อต้องการเวลาแบบ 12 ชั่วโมง

ตัวอย่างนี้กำหนดนาทีทีละ 15 นาที และวินาทีทีละ 10 วินาที

Time Range

เลือกช่วงเวลา

ใช้ range_time: true เพื่อให้มีตัวเลือกเวลาเริ่ม/สิ้นสุดแยกกันตาม format

ถ้าใช้ H:i จะเลือกชั่วโมงและนาที 2 ชุด และถ้าใช้ H:i:s จะมีวินาทีเพิ่มให้อัตโนมัติ

Date

เลือกวันพื้นฐาน

เหมาะกับฟอร์มทั่วไปที่ต้องการเก็บค่าแบบ Y-m-d

ตัวอย่างนี้ใช้ภาษาไทย และเก็บค่าลง input เป็น ค.ศ. ตาม format ที่กำหนด

Buddhist

แยกได้ว่าปฏิทินหรือ input จะเป็น พ.ศ.

ใช้ buddha: true ให้ปฏิทินแสดงปี พ.ศ. และถ้าต้องการให้ช่องที่ผู้ใช้เห็นเป็น พ.ศ. ด้วย ให้เปิด buddha_input: true เพิ่มใน format ที่มีปี

กรณีที่สองจะใช้ mirror input แบบอ่านอย่างเดียว ผู้ใช้เห็นเป็น พ.ศ. แต่ค่าจริงใน source input ที่ใช้ submit ฟอร์มยังเป็น YYYY-MM-DD แบบ ค.ศ.

Datetime

เลือกวันและเวลา

ใช้ format: "Y-m-d H:i" เพื่อแสดง calendar และ time panel พร้อมกัน

รายการนาทีแสดงครบตั้งแต่ 00 ถึง 59 เพื่อให้เลือกเวลาได้ละเอียดทุกนาที

Time

เลือกเฉพาะเวลา

ถ้า format เป็น H:i จะเหลือเฉพาะ time picker

เหมาะกับ use case เวลาเปิด-ปิด, เวลานัด, หรือ slot เฉพาะเวลา

Month

เลือกเฉพาะปีและเดือน

ถ้ากำหนด format: "Y-m" จะเลือกได้แค่ปีและเดือน

ตัวอย่างนี้เปิด พ.ศ. ด้วย เพื่อให้เห็นการทำงานร่วมกันของ month picker และ Buddhist year

Year

เลือกเฉพาะปี

ถ้ากำหนด format: "Y" จะเข้าโหมดเลือกปีโดยตรง

เหมาะกับรายงานสรุป, ปีงบ, หรือ filter ที่ต้องการแค่ปีเดียว

Limits

กำหนดช่วงวัน และปิดเสาร์อาทิตย์

ใช้ min_date, max_date และ disabled_weekdays ได้พร้อมกัน

ตัวอย่างนี้เลือกได้เฉพาะวันจันทร์ถึงศุกร์ ระหว่าง 2026-01-01 ถึง 2026-12-31

Open / Close Rules

ระบุวันเปิดและวันปิดแบบรายวันและเป็นช่วง

ผสม enabled_ranges, disabled_dates และ disabled_ranges ได้

ตัวอย่างนี้เปิดให้เลือกเฉพาะเดือนเมษายนทั้งเดือน และ 1-15 มิถุนายน แต่ปิดวันสงกรานต์กับช่วง 20-22 เมษายน

Function Rules

กำหนดวันเปิดและวันปิดด้วย function

ใช้ enabled_date และ disabled_date เมื่อต้องการ business rule ที่ยืดหยุ่นกว่า list ปกติ

ตัวอย่างนี้ช่วยอธิบายการทำงานของ whitelist + blacklist แบบ function ในเคสจริง

Multiple

เลือกหลายวัน

เปิด multiple: true เพื่อให้ picker เก็บหลายค่า และกำหนด multiple_separator ได้

เลือกซ้ำวันเดิมอีกครั้งเพื่อยกเลิกวันนั้นได้

Inline

แสดง picker แบบ inline

เปิด inline: true เมื่อต้องการให้ปฏิทินแสดงอยู่บนหน้าเสมอ

ตัวอย่างนี้ตั้ง week_start: 1 เพื่อให้สัปดาห์เริ่มวันจันทร์

Native Input

ใช้กับ native input ได้

ถ้าไม่กำหนด format library จะเดาจาก type ของ input ให้

ตัวอย่างนี้ไม่ต้องใส่ format เพราะอ่านจาก type="date" และ type="time"

Data Attributes

ตั้งค่าจาก HTML โดยตรง

ใช้ data-mangopicker-* เมื่ออยากให้ option อยู่กับ input และ init ด้วย selector สั้นๆ

ตัวอย่างนี้ตั้งภาษาไทย, พ.ศ., ปิดวันที่ผ่านมา, เริ่มสัปดาห์วันจันทร์ และแสดงปุ่ม reset จาก attribute ทั้งหมด

Callbacks + Functions

ใช้ callback และ rule function

ตัวอย่างนี้ใช้ function บน window ผ่าน data attributes เพื่อให้ HTML เรียก callback/rule ได้

ยังไม่ได้เลือกวันรับสินค้า
Render Hook

render_cell_date สำหรับแต่ง day cell

ใช้ render_cell_date(detail) เมื่อต้องการเปลี่ยนข้อความหรือ HTML ภายในช่องวันแต่ละช่อง

ตัวอย่างนี้คืน HTML string จาก function เพื่อใส่ label ย่อยใน day cell ตามวันในสัปดาห์

Callbacks

ดู callback ทำงานจริง

การ์ดนี้ต่อ callback ทั้ง on_open, on_close, on_select, on_change และแสดงผล event log ให้เห็นทันที

log ด้านขวาจะเพิ่มตามการใช้งานล่าสุด และเก็บค่าไว้ 8 รายการล่าสุด

ยังไม่มี event
ลองคลิก input ด้านซ้ายเพื่อเริ่มทดสอบ callback
API

เรียกใช้งานผ่าน manager API

ใช้ open(), close(), set_value(), get_value(), clear(), reset() ได้จาก object ที่ init() คืนกลับมา

ผลลัพธ์ API
กดปุ่มด้านซ้ายเพื่อเรียกใช้งาน manager ที่ได้จาก window.mangoPicker.init(...)

Callbacks

Callback เรียกเมื่อไหร่ หมายถึงอะไร ตัวอย่าง
on_open(detail)เปิด pickerใช้รู้ว่าผู้ใช้เริ่มเปิด picker แล้วon_open: function (detail) { console.log(detail.value); }
on_close(detail)ปิด pickerใช้ตรวจสถานะหลังผู้ใช้ออกจาก pickeron_close: function (detail) { console.log(detail.values); }
on_select(detail)เลือกวัน/เดือน/ปี/เวลาใช้ preview ค่าระหว่างเลือกon_select: function (detail) { console.log(detail.display_value); }
on_change(detail)ค่า input เปลี่ยนจริงใช้ sync ฟอร์ม, validate หรือเรียก APIon_change: function (detail) { console.log(detail.value); }

Function Options

Function option หมายถึงอะไร ต้อง return ตัวอย่าง
enabled_date(candidate_date)กำหนดว่าวันไหน "เลือกได้เท่านั้น"true เพื่ออนุญาตวันนั้นenabled_date: function (date) { return date.getDay() !== 0; }
disabled_date(candidate_date)กำหนดว่าวันไหนต้องปิดไม่ให้เลือกtrue เพื่อปิดวันนั้นdisabled_date: function (date) { return date.getDate() === 13; }
render_cell_date(detail)กำหนด text/HTML ภายใน day cell แต่ละช่องHTML string หรือ text ที่อยากให้แสดงrender_cell_date: function (detail) { return "<span>" + detail.day + "</span>"; }

render_cell_date(detail) จะถูกเรียกทุกครั้งที่ render ปฏิทิน และใน detail จะมีข้อมูลอย่าง date, day, display_year, is_today, is_selected ให้ใช้ตัดสินใจง่ายขึ้น
